# Roadmap

## Phase 0: 项目准备

- 确定只做本地模拟，不实盘下单。
- 确定默认本金、手续费、滑点、交易单位。
- 固化配置文件格式。
- 建立日志和报告目录。
- 确定本地技术栈：FastAPI、SQLite、DuckDB、Parquet、React、K线图库。

交付物：

- 项目大纲。
- 示例配置。
- 策略规格说明。
- 技术栈说明。

## Phase 1: 最小数据和模拟闭环

目标：能拉取 ETF 行情，写入本地存储，并跑一次完整的历史模拟。

任务：

- 接入 AkShare ETF 日线数据。
- 将标准化 OHLCV 行情写入 Parquet。
- 用 DuckDB 查询本地 Parquet 行情。
- 读取配置文件。
- 自动筛选 ETF。
- 初始化虚拟账户。
- 实现网格做T规则。
- 模拟买入、卖出、手续费和滑点。
- 将信号、成交、持仓、账户快照写入 SQLite。
- 输出 `signals.csv`、`trades.csv` 和 `daily_summary.csv`。

验收标准：

- 能用 100000 CNY 假设本金跑指定日期区间。
- 任何交易都有触发原因。
- 不出现现金为负、卖出超过持仓、非 100 份整数倍等错误。
- 断网后仍能读取本地缓存做回测。

## Phase 2: 本地 API 和 K 线看板

目标：能在浏览器里查看 ETF K 线、交易标记和模拟账户状态。

任务：

- 用 FastAPI 提供本地接口。
- 用 React + Vite 建立前端项目。
- 用 TradingView Lightweight Charts 展示 K 线和成交量。
- 在 K 线上叠加模拟买卖点。
- 用 ECharts 展示账户净值、收益率和回撤。
- 展示当前候选 ETF、持仓、现金、交易日志和风控拦截原因。

验收标准：

- 启动后端和前端后，可以在浏览器查看本地模拟结果。
- 切换 ETF 时，K 线、信号和交易标记同步更新。
- 前端只读展示，不提供真实账户登录或下单能力。

## Phase 3: 更真实的做T模拟

目标：更贴近 A 股 ETF 交易限制。

任务：

- 加入 T+1 可卖份额限制。
- 加入底仓保护。
- 加入单日交易次数限制。
- 加入单只 ETF 仓位上限。
- 支持分钟线模拟。
- 支持多个 ETF 同时运行。

验收标准：

- 可以比较日线和分钟线下的策略差异。
- 能输出被风控拦截的信号。

## Phase 4: 策略比较和参数扫描

目标：验证参数稳定性，而不是只找最优结果。

任务：

- 加入均值回归做T策略。
- 支持网格间距、止盈比例、单次金额参数扫描。
- 输出收益、最大回撤、胜率、交易次数、手续费占比。
- 对每只 ETF 生成策略适配评分。

参考：

- backtesting.py：简单回测结构。
- vectorbt：参数扫描思想。

## Phase 5: 实时模拟提醒

目标：盘中准实时运行，但仍然只模拟和提醒。

任务：

- 定时拉取 ETF 实时行情。
- 实时更新虚拟账户。
- 触发信号时输出提醒。
- 支持控制台、桌面通知、邮件或 Telegram 推送。

验收标准：

- 连续运行一整天不中断。
- 断网或接口失败时不会乱生成交易。
- 交易日志可复盘。

## Phase 6: 存储和性能升级

目标：在数据量和运行频率上来后，再升级存储和热点模块。

任务：

- 根据实际数据量评估是否引入 MongoDB 保存策略事件和复杂日志。
- 根据实际瓶颈评估是否把撮合、风控或行情处理热点改成 C++/Rust。
- 加入更稳定的后台任务调度。
- 优化分钟线和 tick 级数据读取。

## 暂不做

- 不接真实券商账户。
- 不自动实盘下单。
- 不做高频交易。
- 不做收益保证。
- 不保存任何敏感登录信息。
